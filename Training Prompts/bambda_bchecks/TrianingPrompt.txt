════════════════════════════════════════════════════════════════
SECTION 1: BAMBDA JSON FORMAT
════════════════════════════════════════════════════════════════

Every Bambda is a JSON object with EXACTLY these fields:
{
"id": "<uuid>",
"name": "<name>",
"description": "<description>",
"function": "<FUNCTION_TYPE>",
"location": "<LOCATION>",
"source": "<java code as escaped string>"
}

CRITICAL FIELD NAMES:

- The field is "function" NOT "functionType"
- The field is "source" NOT "script"
- Getting these wrong causes: "function is required, location is required" error

════════════════════════════════════════════════════════════════
SECTION 2: VALID function + location COMBINATIONS
════════════════════════════════════════════════════════════════

ONLY use these exact combinations. Never invent your own values.

| User says | "function" value | "location" value |
| --- | --- | --- |
| proxy http / filter | VIEW_FILTER | PROXY_HTTP_HISTORY |
| proxy websocket | VIEW_FILTER | PROXY_WEBSOCKET |
| match replace request | MATCH_AND_REPLACE_REQUEST | PROXY_HTTP_HISTORY |
| match replace response | MATCH_AND_REPLACE_RESPONSE | PROXY_HTTP_HISTORY |
| logger / logger view | VIEW_FILTER | LOGGER |
| logger capture | CAPTURE_FILTER | LOGGER |
| sitemap / target | VIEW_FILTER | SITEMAP |
| repeater / custom action | CUSTOM_ACTION | REPEATER |
| custom column | CUSTOM_COLUMN | PROXY_HTTP_HISTORY |
| scanner / active scan | SCAN_CHECK_ACTIVE_PER_REQUEST | SCANNER |

MISTAKES TO NEVER MAKE:
✗ "functionType": "PROXY_HTTP_FILTER"   → WRONG field name AND wrong value
✗ "script": "..."                       → WRONG field name, must be "source"
✗ "location": "HTTP_HISTORY"            → WRONG, must be "PROXY_HTTP_HISTORY"
✗ "location": "TARGET_SITE_MAP"         → WRONG, must be "SITEMAP"
✗ "location": "PROXY"                   → WRONG, must be "PROXY_HTTP_HISTORY"
✗ "function": "MATCH_AND_REPLACE"       → WRONG, must specify REQUEST or RESPONSE
✗ "location": "WEBSOCKETS_HISTORY"      → WRONG, must be "PROXY_WEBSOCKET"

════════════════════════════════════════════════════════════════
SECTION 3: CONTEXT OBJECTS PER LOCATION
════════════════════════════════════════════════════════════════

Each location gives you different context objects. Use the correct one.

PROXY_HTTP_HISTORY (VIEW_FILTER):

- Context: requestResponse
- Returns: boolean
- Example: return requestResponse.hasResponse() && requestResponse.response().statusCode() == 200;

PROXY_WEBSOCKET (VIEW_FILTER):

- Context: message
- Returns: boolean
- Example: return message.payload().length() > 0;

MATCH_AND_REPLACE_REQUEST:

- Context: requestResponse
- Returns: modified HttpRequest object (NOT boolean)
- Example: return requestResponse.request().withAddedHeader("X-Custom", "value");

MATCH_AND_REPLACE_RESPONSE:

- Context: requestResponse
- Returns: modified HttpResponse object (NOT boolean)
- Example: return requestResponse.response().withRemovedHeader("X-Powered-By");

LOGGER (VIEW_FILTER):

- Context: requestResponse
- Returns: boolean
- Has access to: ToolType.*, HighlightColor.*
- Example: requestResponse.annotations().setHighlightColor(HighlightColor.RED); return true;

LOGGER (CAPTURE_FILTER):

- Context: requestResponse
- Returns: boolean

SITEMAP (VIEW_FILTER):

- Context: node ← CRITICAL: NOT requestResponse
- Returns: boolean
- Example: return node.requestResponse().hasResponse();
- NEVER use requestResponse directly in sitemap scripts

REPEATER (CUSTOM_ACTION):

- Context: requestResponse, httpEditor, api(), logging()
- Returns: void (no return value)
- Send requests with: api().http().sendRequest(...)
- Log with: logging().logToOutput("message")
- Example: var req = requestResponse.request(); logging().logToOutput(req.url());

CUSTOM_COLUMN (PROXY_HTTP_HISTORY):

- Context: requestResponse, utilities
- Returns: String (NOT boolean)
- Example: return requestResponse.request().pathWithoutQuery();
- JSON utils: utilities.jsonUtils().readString(body, "fieldName")

SCANNER (SCAN_CHECK_ACTIVE_PER_REQUEST):

- Context: requestResponse, http, api()
- Returns: AuditResult (NOT boolean)
- Always check: if (!requestResponse.hasResponse()) { return null; }
- Send requests: http.sendRequest(...) OR api().http().sendRequest(...)
- Return issues: return AuditResult.auditResult(AuditIssue.auditIssue(...));
- Return clean: return AuditResult.auditResult();

SECTION 5: CODING RULES
════════════════════════════════════════════════════════════════

1. The source is Java code (not Kotlin, not JavaScript)
2. No import statements — all Burp API classes are pre-imported
3. For VIEW_FILTER and CAPTURE_FILTER → always return boolean
4. For CUSTOM_COLUMN → always return String, never null (return "" instead)
5. For CUSTOM_ACTION → void, never write a return statement with a value
6. For MATCH_AND_REPLACE_REQUEST → return the request object, never boolean
7. For MATCH_AND_REPLACE_RESPONSE → return the response object, never boolean
8. For SCAN_CHECK_ACTIVE_PER_REQUEST → return AuditResult, never boolean
9. Always guard response access: if (!requestResponse.hasResponse()) { return false; }
(or return null for scanner, or return resp for match-replace)
10. In SITEMAP scripts use node.requestResponse() NOT requestResponse directly
11. In PROXY_WEBSOCKET scripts use message NOT requestResponse
12. Escape all double quotes in source as \" and all backslashes as \\
13. Newlines in source must be \n (the JSON string must be single-line)

**══════════════════════════════════════════════════════════════SECTION 8: MONTOYA API GOTCHAS ══════════════════════════════════════════════════════════════**

**1. NO `request.cookies()`:** Montoya `HttpRequest` does NOT have a `.cookies()` method. You MUST use:
`request.parameters().stream().filter(p -> p.type() == HttpParameterType.COOKIE)`

**2. TYPE SAFETY:**
When using `.toList()` or `.stream()`, ensure you reference the parameter correctly. Montoya parameters use `.name()` and `.value()`.

**3. JWT/BASE64:**
Use `utilities.base64Utils().decode(string).toString()` to decode parts.

**4. STRINGS:**
Always return an empty string `""` in CUSTOM_COLUMN if no value is found. NEVER return `null`.
